Vibe Coding初探

发表于 2026-02-19 更新于 2026-02-20 分类于 TECH 阅读次数： 本文字数： 893 阅读时长 ≈ 3 分钟

前几天在老家没网，闲着没事就刷视频，不刷不知道，一下子给我打开了新世界的大门

摸索

一开始刷到的是跟AI Agent相关的东西，最早是看到有人用claude code调电机pid，居然调着调着自己就调平了，当时我就觉得卧槽这agent现在发展到这个程度了？虽然agent这概念也出了挺久了我也听了挺久的，但人可能就是有惯性，一开始chatgpt到gemini免费会员用习惯了，感觉也挺好用的，面对网上铺天盖地的vibe coding没怎么在意，看了几个视频越来越觉得要跟上脚步了，不管vibe coding的前景怎么样 会不会对我的基础造成什么影响 这个效率提升是真恐怖啊!

好不容易熬到了回家，终于可以畅快探索了。最开始在gemini网页版的帮助下，弄了cline和openrouter，这里简单解释一下：
cline，claude code ，codex，copliot等都是agent，可以单独作为cli agent，也可以作为vscode插件，cline和copilot可以支持各种模型，claude code和codex应该是只支持自家的
然后还有cursor，这个是一个vscode的魔改版，订阅制，最低20￥/month，只能用最高端的几个模型，比如claude-opus-4.6 codex5.3之类的 我没用过
其实在现在这个时间点 用上最新的模型还挺简单的 走正规路线，那就是使用openrouter

价格透明，和各家厂商的官方api都一个价，但openrouter收取5.5%的服务费(最低0.8$),cline也可以用,但提供的模型少一些,有免费的也有高端的,也是官方价,服务费略低一些

体验

用cline提供的免费模型minimax/minimax-m2.5试着写了个简单的小工具,检测hid设备

本来我以为国产厂商的这种免费模型应该不怎么样的，但是试用了一下，居然效果还不错，编译报错了几次之后成功写出了可用的程序。

新世界的大门

直到以上这些其实都还行,我觉得vibe coding还是有些门槛的，并且也有些烧钱。但很快我发现，居然还有一个东西叫api中转商！不同于openrouter直接原价使用api，这些中转站通过一些逆向客户端或者一些其他ai相关的产品，可以以极其低廉的价格搞到高端模型的api，可以把价格打到原厂的几十分之一，甚至还有免费的公益站！甚至中转站已经在我不知道的很长一段时间里发展得非常完善了，还有自己的通用网站面板new-api(类似机场的sspanel)，各种各样的中转站层出不穷，甚至只要站长想的话，也完全可以用什么拉新拿佣金，签到得积分之类的方法留住客户，高端ai的使用落实到应用居然变成了这么灰色地带的神奇东西…..真是太难以置信了。

不过好消息是，虽然不是正经路子，但我们确实用得起昂贵的模型了。在公益站的帮助下，我顺利地写了一点小项目，虽然还有些bug，但整体是非常满意了。
